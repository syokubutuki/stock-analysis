## 1. 問題設定 — (a) と (b) の厳密な分離

ここを曖昧にすると検証が空回りする。明確に別物として切り分ける。

| | (a) 曜日リターンアノマリー | (b) 動力学的な週内アトラクタ（本書の主眼） |
|---|---|---|
| 対象 | リターンの **1次モーメント**（平均・分布） | 位相空間上の **軌道の幾何** |
| 主張 | 「月曜の平均リターンが低い」など | 「状態ベクトルが 月→火→…→金 で位相空間の特定領域を **周期5で巡回** する」 |
| 数理 | 曜日条件付き期待値 E[r \| dow] の差 | 週次位相に同期した **リミットサイクル / トーラス** の存在 |
| 既存実装 | `曜日別リターン分布`（直近コミット） | 未実装（本書） |

(b) は (a) より強い主張である。(a) は「平均が違う」だけだが、(b) は **軌道そのものが曜日で組織化されている**（=決定論的な週内サイクルの骨格がある）ことを意味する。

(b) が成立すれば (a) は系として導かれることが多いが、逆は成り立たない。平均が曜日で違っても（=(a)あり）、軌道が位相空間でランダムに散っていれば (b) はない。**狙うのは「軌道の幾何としての週次位相ロック」**。

## 2. 動力学モデル — 週次クロックで周期駆動された力学系

仮説として、株価リターン系列を以下の周期駆動された確率力学系とみなす。

```
状態:        x(t) ∈ R^m  （Takens遅延座標で再構成）
週次位相:     φ(t) = (営業日インデックス mod 5)  ∈ {0,1,2,3,4}  （月=0 … 金=4）

dx/dt = F(x, φ(t)) + ノイズ
        F が φ に依存する  ⇔  週次クロックで周期駆動されている
```

- **位相ロックあり**: 拡張状態空間 (x, φ) において、φ の各値に対して x が特定の領域に引き寄せられる。trajectory は (x, φ) 上で **周期5のリミットサイクル / トーラス** を描く。
- **位相ロックなし**: F が φ に依存しない（F(x, φ) = F(x)）。x の分布は φ と独立。

検証とは要するに「**F の φ 依存性が、ノイズ・自己相関だけで説明できる以上に有意か**」を判定すること。

## 3. 検証方法（数式つき）

既存ライブラリを活かせる順・力学系として素直な順に並べる。

### 3.A 曜日層別アトラクタ（`AttractorExplorer` 拡張）

Takens埋め込みベクトル
```
v(t) = ( r(t), r(t-τ), r(t-2τ), … , r(t-(m-1)τ) ) ∈ R^m
```
の各点を曜日 φ(t) で色分けする。

各曜日の **重心** と **曜日内分散**:
```
μ_k = (1/n_k) Σ_{t: φ(t)=k} v(t)            （曜日 k の重心）
S_within = Σ_k Σ_{t: φ(t)=k} ‖v(t) − μ_k‖²   （曜日内平方和）
S_between = Σ_k n_k ‖μ_k − μ̄‖²              （曜日間平方和, μ̄ は全体重心）
```

**位相ロック統計量（位相空間版 一元配置分散分析の F 比）**:
```
PL = [ S_between / (K−1) ] / [ S_within / (N−K) ]    （K=5, N=総点数）
```
- PL が大きい ⇔ 同一曜日が位相空間で塊になる ⇔ 週次位相ロック。
- ただし生の F 値は自己相関で嵩上げされるため、**必ず §4 のサロゲートと比較**。

**巡回性**: 重心 μ_0 → μ_1 → … → μ_4 → μ_0 が位相空間で **環状に順序よく並ぶ** なら、それがリミットサイクルの骨格。重心パスの総経路長・自己交差・周回方向を可視化する。

### 3.B 週次ストロボ写像（ポアンカレ断面）— 力学系として最もクリーン

周期駆動系の位相ロック判定の定石。**毎週同じ曜日で1点だけ** サンプリングする（ストロボ＝周期駆動周波数でのスナップショット）。

```
固定曜日 k* について:
  P_k* = { v(t) : φ(t)=k* }     （週次ストロボ写像 / ポアンカレ断面上の点列）
```
- 週次サイクルがクリーンなリミットサイクルなら、P_k* は **小さな塊に収束**（週次リターン写像の不動点）。
- 散らばるなら位相ロックなし。
- τ・m の恣意性に比較的頑健で、「週周期が骨格か」を一発で見られる。
- 収束度の定量化: P_k* の分散 / 全点の分散（比が小さいほど不動点性が強い）。

### 3.C リカレンスのラグ構造（既存RQA）

リカレンス行列
```
R(i,j) = Θ( ε − ‖v(i) − v(j)‖ )      （Θ: ヘヴィサイド, ε: 近傍半径）
```
について、**ラグ ℓ = |i−j| ごとのリカレンス率**を集計:
```
RR(ℓ) = (1/(N−ℓ)) Σ_i R(i, i+ℓ)
```
- 週内アトラクタは RR(ℓ) が **ℓ = 5, 10, 15 にピーク** を出す（週次再帰）。
- 既存 `computeRecurrencePlot` の対角線構造から抽出可能。

### 3.D Lomb-Scargle 周期検定（既存 `lomb-scargle.ts`）

週末欠損（不等間隔）に強い LS ペリオドグラムで、**周期 5 営業日（周波数 f = 0.2 cycle/day）にピーク** が立つか検定。
- LS パワーの有意性は、ホワイトノイズ帰無仮説の解析的閾値、または §4 のサロゲートで判定。
- 周波数領域からの裏取りであり、3.A/3.B（時間・位相領域）と独立な証拠になる。

### 3.E 条件付き予測力（既存 `SimplexPredictionChart`）

埋め込み座標に **週次位相を追加座標** として加える:
```
ṽ(t) = ( v(t),  cos(2π φ(t)/5),  sin(2π φ(t)/5) )
```
（φ を円周上に埋め込むことで「月と金が隣」のような不連続を避ける）

Simplex / S-map の予測スキル（相関係数 ρ または MAE）を
- 「v(t) のみ」 vs 「ṽ(t)（位相つき）」
で比較。**位相つきで有意に改善** すれば、週内構造が *予測に効く情報* を持つ＝実利あり。改善ゼロなら、週内構造があってもトレードには使えない。

## 4. サロゲート検定 — 自己欺瞞を防ぐ要

§3 の統計量（PL, RR(5), LS パワー, 予測スキル差）は **単なる自己相関でも見かけ上出る**。比較対象（帰無分布）を必ず構築する。

| サロゲート | 破壊するもの | 保持するもの | 検出対象 |
|---|---|---|---|
| **曜日ラベルシャッフル** | 曜日整合 | 自己相関・分布 | 「週次位相ロックは自己相関では説明できない」 |
| 位相ランダム化 (FT surrogate) | 非線形構造 | パワースペクトル | 線形性 vs 非線形性 |
| ブロックブートストラップ | 長周期構造 | 短期自己相関 | 構造の頑健さ |

**手順**:
1. 実データで統計量 T_obs を計算。
2. サロゲートを B 回（例: 999回）生成し、各々で T_b を計算。
3. p値 = (1 + #{T_b ≥ T_obs}) / (B + 1)。
4. p < 0.05（多重検定なら Bonferroni / FDR 補正後）で「週内構造あり」。

**多重検定の注意**: 曜日 5 本 × 複数統計量 × 複数銘柄 を検定すると偽陽性が膨らむ。サロゲートベース p 値 + FDR 補正を既定とする。

## 5. 限界と前提条件

実装・解釈時に常に意識する制約。

1. **周期5は分解能ギリギリ**。`m=3, τ=1` で既に t〜t-2 を食う。週周期を捉えるには `m·τ ≈ 5` になるよう τ を選ぶ必要があり、`AttractorExplorer` のデフォルトでは週内ループは潰れて見えない。→ 本コンポーネントでは τ の既定値・推奨レンジを週内構造向けに設定する。

2. **既知カレンダー要因との交絡**。月曜効果・週初/週末効果・SQ・先物ロール・米雇用統計（第1金曜）等。決定論的アトラクタと外生カレンダーイベントの区別は原理的につかない。「動力学的」と言っても *外生周期駆動* である可能性が高い（むしろ §2 のモデルはそれを前提にしている）。

3. **非定常**。曜日効果は数十年スケールで減衰・反転する。**必ずローリングで** 検証する（全期間集計は誤判定の元）。流動性の高い指数では概ね裁定済みのことが多い。

4. **期待値の置き方**。狙うのは「強い単独アルファ」ではなく「**弱いが安定したバイアス／実行タイミングの最適化／リスクの曜日季節性**」。リターン季節性よりボラ季節性の方が持続的。

5. **サンプル数**。週次ストロボ写像は 1 週 1 点なので、10年日足でも約 520 点。曜日層別も各曜日 N/5。高次元埋め込みは統計的に苦しい → m は小さく（2〜3）保つ。

## 6. トレードへの取り込み

構造がサロゲート（§4）＋アウトオブサンプルを通った場合のみ。頑健な順。

### 6.1 位相条件付きリスク（最も堅い）
曜日内分散 S_within,k が体系的に高い曜日があれば、**曜日でポジションサイズ／ストップ幅をスケール**（週内ボラ季節性）。
- リターン季節性より *ボラ季節性* の方が持続的（例: 月曜ギャップリスク）。
- ポジションサイズ ∝ 1 / σ_k（曜日 k の条件付きボラ）。

### 6.2 実行タイミングの最適化
アルファ生成より「**既に決めた建玉をどの曜日で執行/リバランスするか**」。
- 高ボラ曜日の取引を避け、サイクルの谷の曜日に執行を寄せる。
- 取引コスト・スリッページの曜日季節性も併せて評価。

### 6.3 位相条件付き方向バイアス
`KramersMoyalChart` のポテンシャル V(p) を **曜日で条件付け** し、谷（サイクル始点）の曜日で積み増し、ピーク曜日で軽くする。
- 主シグナルへの **±の小さな修飾子** に留める（単独アルファとしては弱い前提）。

### 6.4 メタゲート
既存ドキュメントの「予測スキル＝メタ指標」と同じ思想。
- **ローリング位相ロック統計量 PL(t) がサロゲート閾値を超えている期間だけ** 曜日チルトを有効化。
- PL(t) が減衰したら無効化。構造の非定常性（§5-3）への対処そのもの。

### トレードシグナルまとめ

| 指標 | シグナル | 解釈 | アクション |
|---|---|---|---|
| PL（位相ロック比） | サロゲート閾値超 | 週次位相ロックが有意 | 曜日チルト有効化 |
| ストロボ収束度 | 高 | 週次リターン写像が不動点的 | 週次サイクル予測を活用 |
| σ_k（曜日内ボラ） | 曜日で偏在 | ボラの週内季節性 | サイズ/ストップを曜日でスケール |
| 重心パス位相 | 谷の曜日 | サイクル始点 | 積み増し |
| 重心パス位相 | ピークの曜日 | サイクル頂点 | 軽量化/利確 |
| 位相つき予測スキル改善 | 有意 | 週内構造が予測に効く | アクティブに活用 |
| 位相つき予測スキル改善 | ゼロ | 構造はあるが予測不能 | 6.1/6.2 のみ（方向は張らない） |

### データ・実装上の注意
- **営業日インデックス mod 5**: 祝日で曜日と営業日インデックスがずれる。「曜日（実カレンダー）」と「営業日位相（直前営業日からの連番 mod 5）」のどちらを位相とするか選択式にする。動力学的には後者（営業日位相）がモデルに忠実だが、外生カレンダー要因の検証には前者（実曜日）も要る。**両方サポート**し比較できるとよい。
- `time` 文字列から曜日を引く際のタイムゾーン注意（Yahoo データの日付解釈）。
- 週末ギャップ（金→月）はオーバーナイトリターンの特殊ケース。`SeriesMode` の overnight/intraday と組み合わせると週内構造の発生源（場中 vs 持ち越し）を切り分けられる。

## 8. 将来拡張

- **マルチ銘柄の位相同期**: 複数銘柄の週次位相ロックが揃うか（市場全体の週次クロック vs 個別要因）。クラマス位相同期指標（Kuramoto order parameter）。
- **適応的位相**: 周期を5に固定せず、瞬時位相（Hilbert変換）から推定。SQ週・月末など準周期構造への一般化。
- **月内・四半期内アトラクタ**: 同じ枠組みを周期21（月）・63（四半期）へ。ターン・オブ・ザ・マンス効果の動力学版。
- **イベント位相**: カレンダー曜日でなく、決算・FOMC・SQ からの経過日を位相に。外生イベント駆動アトラクタ。
- **状態依存の位相ロック**: PL をボラレジーム（高ボラ/低ボラ）で層別。位相ロックがレジーム依存かを検証。

- 散布・重心は埋め込み第1・第2成分への 2D 射影で表示(3D 回転 UI は未実装。dim=3 でも PL は3次元で計算)。
- weekdayIndex は `getUTCDay()` で TZ ずれを回避。週末(calendarで-1)は除外。
- サロゲートは毎 useMemo で `Math.random()` 抽選のため再計算で僅かに揺れる(p値は安定)。「サロゲート再抽選」ボタンあり。シード固定は未対応。
- 全期間集計のため非定常性は未対処 → フェーズ3 のローリング PL(t) が必要(§6.4 メタゲートの前提)。
- 計算量は B=499 × O(N·m) で日足10年でも軽く、Worker 化は未実施(フェーズ2以降で重くなれば検討)。
- `tsc --noEmit` パス。

### フェーズ2(裏取り) 実装済み (2026-06-11)

**追加シグネチャ**(同 `weekly-phase-attractor.ts`)
```ts
computeRecurrenceLagProfile(prices, seriesMode,

computeWeeklySpectrum(prices, seriesMode): WeeklySpectrumResult
```

### フェーズ3(非定常監視 / メタゲート) 実装済み (2026-06-11)

**追加シグネチャ**
```ts
computeRollingPL(prices, seriesMode,
  { tau, dim, phaseMode, window?=252, step?=21, threshold }): RollingPLResult
```
- 窓(既定252日)ごとに `fRatio` で PL を計算、`step`(既定21日)でスライド。
- **閾値は全期間サロゲートの95%分位点**(`computeWeeklyPhaseAttractor` の `surrogateQ95` を渡す)。窓ごとサロゲートは計算重のため未採用 → 簡易メタゲート。

### フェーズ3-E(位相つき予測 = トレード価値の最終確認) 実装済み (2026-06-11)

**追加シグネチャ**
```ts
computePhaseAugmentedSimplex(prices, seriesMode,
  { tau, dim, phaseMode, phaseWeight?=1, nShuffle?=20 }): PhaseSimplexResult
```
- 埋め込み座標を z-score 標準化 → 週次位相を円周埋め込み `(w·cos(2πφ/5), w·sin(2πφ/5))` で追加(φの月↔金不連続を回避)。
- 自前の軽量 Simplex(`simplexCore`: 前70%をライブラリ、k=m+1、距離加重)で **ベースライン vs 位相つき** の予測スキル ρ(Pearson)と方向的中率を比較。
- **位相シャッフル対照**(位相ラベルをシャッフルして同じ拡張、nShuffle=20平均)を計算。`improves = ρ_aug > ρ_base かつ ρ_aug > ρ_shuffled + 0.005`。
- 設計判断: 座標を1本足せばノイズでも僅かに ρ が動くため、シャッフル対照を超えて初めて「効く」と判定。UI に重みスライダー(0〜3)で感度確認。
- 既存 `attractor-investment.ts` の simplex を使わず自前実装した理由: 位相座標を距離計算に組み込むカスタム埋め込みが必要で、既存APIは生系列前提のため。

### 残り候補 実装/判断 (2026-06-12)

**曜日条件付き KM — 実装済み**(`computeWeeklyPhaseKM(prices, phaseMode)`)
- 設計変更: 価格条件付け(§6.3 の文言)はデータ5分割で脆弱なため、**週次位相 φ を状態変数とした KM** として実装。
- 日次対数リターンを増分とし(seriesMode 非依存)、曜日別ドリフト μ(φ)=E[r\|曜日]・拡散 σ(φ)=std(r\|曜日)・月→金の累積経路を算出。
- 出力: `entryPhase`(累積の谷=積み増し), `exitPhase`(ピーク=軽量化), `highVolPhase`/`lowVolPhase`(§6.1 リスク季節性)。UI はテーブル+トレード解釈。

**窓ごとサロゲートによる厳密なローリング有意性 — 実装済み**(`computeRollingPL` を改修)
- 各窓で曜日シャッフルサロゲート(B=99)を実行 → 窓ごと95%閾値・片側p値・`significant`。
