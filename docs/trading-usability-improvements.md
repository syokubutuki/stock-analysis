# トレード判断への適用性を高める改善案

作成日: 2026-06-23

本ドキュメントは、既存の分析機能を「より投資判断に使いやすく」するための改善案をまとめたもの。
**「任意日付レンジ＋グローバル曜日マスク」は別途着手予定のため本書からは除外**し、それ以外の
データ表示・チャート・UI・自由度の改善を、実装の足場（既存ファイル）つきで優先度順に整理する。

関連ファイル（現状の足場）:
- `app/hooks/useAnalysisData.ts` — `period` / `filteredPrices`（末尾固定スライス）
- `app/lib/conditional-forward-returns.ts` — 状態→フォワードリターン集計エンジン（FDR・ブートCI・年次符号一致を実装済）
- `app/components/analysis/CustomReturnChart.tsx` — 任意エントリー/エグジット累積リターン＋B&H比較＋日付ピッカー
- `app/lib/stats-significance.ts` — t検定 / Benjamini-Hochberg FDR / ブロックブートCI

## 1. 条件バケットのユーザー編集化 ★★★

### 現状の制約
`conditional-forward-returns.ts` の `buildStateFn`（`:181-473`）は強力だが、バケット境界が
**コードにハードコード**されている。例:
- RSI(14): `<30 / 50 / 70`（`:184`）
- ボラレジーム: 標本の3分位固定（`:198-199`）
- 200日線乖離: `±10%`固定（`:214`）

ユーザーが自分の閾値感覚で検証できない。

### 改善内容
- **ビン境界スライダー**: `buildStateFn(prices, axis)` を `buildStateFn(prices, axis, config)` に拡張し、
  各軸の境界配列・ビン数を引数化。UIに数値入力/スライダーを追加。
- **分位数モード切替**: 「固定閾値」⇄「等頻度N分位（quantile）」をトグル。等頻度にすると各バケットの
  サンプル数が揃い、`tTest`→`benjaminiHochberg`（`:563-569`）の検出力が安定する。
- **ビン数可変**: 3/4/5/10分位を選択可能に。`terciles`（`:174-179`）を汎用 `quantiles(vals, k)` に一般化。

### 実装の足場
```ts
// conditional-forward-returns.ts
export interface AxisConfig {
  mode: "fixed" | "quantile";
  thresholds?: number[];   // fixedモード: 境界値
  bins?: number;           // quantileモード: 分割数
}
export function buildStateFn(prices: PricePoint[], axis: StateAxis, cfg?: AxisConfig): StateFn
```
`stateOf` 内の固定 if 分岐を `cfg.thresholds` ベースのループに置換。
`order` ラベルも境界から動的生成（例: `"<30"` → `"<${t[0]}"`）。

### 注意点
- ビンを細かくするとサンプル数が減り過学習・偽陽性が増える。**各バケットに最小N（例: 20）警告**を表示。
- 分位境界はインサンプル全体で決めると弱い先読みになる。厳密にやるなら walk-forward 分位（拡張ウィンドウ）も将来オプション化。

## 2. 「現在地サマリー」意思決定バナーの標準化 ★★★

### 現状の制約
データは豊富だが「**で、今日どうなのか**」への即答が弱い。条件付き分析の多くは
`nowLabel`（`:601-605`）/ `byYear`（`:582`）を既に算出済みなのに、表示で活かしきれていない。

### 改善内容
条件付き分析コンポーネント共通の上部バナー部品 `<NowVerdictBanner>` を新設し、以下を1〜2行で提示:
- **現在のバケット**（`nowLabel`）と、その期待フォワードリターン・勝率・有意性バッジ（`significant`）・信頼区間（`ciLow/ciHigh`）
- **百分位ゲージ**: 現在のRSI/ボラ/乖離が過去分布の何%タイルにいるか
- **再現性**: 「過去同条件 N 回」「直近5年の年次符号一致 m/5」（`byYear` から算出）
- **ベースライン比較**: `baselineMean`/`baselineWin`（`:611-613`）に対する超過

### 実装の足場
`ForwardResult` は必要な値をほぼ持っている。新規計算はほぼ不要で**表示の標準化が主**。
```tsx
<NowVerdictBanner result={forwardResult} />
// 内部で result.buckets.find(b => b.label === result.nowLabel) を参照
```

### 期待効果
「今このRSI帯・このボラ帯にいるとき、過去の同条件では次5日がどうだったか」を画面上部で即断できる。

## 3. 任意リターンビン × 状態軸ヒートマップの一般化 ★★

### 現状の制約
`twoFactorForward`（`:488-521`）は2つの状態軸のクロス集計まで実装済だが、
**フォワードリターン自体を任意ビンに切る**機能がない。

### 改善内容
- フォワードリターンを「±1%刻み」「四分位」「ユーザー指定エッジ」で区切り、
  **状態軸 × リターンビンの頻度/期待値ヒートマップ**を出す。
- 「ある状態のとき、次N日リターンの分布がどの帯に偏るか」を可視化（単なる平均より情報量が多い）。
- N（ホライズン）スライダーと連動し、N=1/5/10/21日で偏りの変化を観察。

### 実装の足場
`twoFactorForward` の `key = ${lx}||${ly}` を `key = ${stateLabel}||${returnBinLabel}` に置換した
新関数 `stateByReturnBin(prices, state, horizon, binEdges)` を追加。描画は既存のヒートマップ部品を流用。

### 注意点
リターンビンは**将来情報**なので「予測」ではなく「条件付き分布の記述」であることをガイドに明記。

## 4. チャート上クロスフィルタ＆条件発生マーカー ★★

### 改善内容
- **ドラッグ選択 → 期間反映**: メイン価格チャートでドラッグした範囲を分析対象期間に反映。
  `CustomReturnChart.tsx:330` で `subscribeVisibleLogicalRangeChange` の実装実績あり、これを横展開。
- **条件発生マーカー**: 価格チャートに「条件成立日」を `createSeriesMarkers(series, markers)`
  （lightweight-charts v5 API）でオーバーレイ。シグナルが実際いつ点灯したかを視覚化。
- **直近シグナルの強調**: 最新の条件成立日とその後の実現リターンをツールチップ表示。

### 注意点
- v5 では `setMarkers()` は廃止、`createSeriesMarkers` を使う（メモリ記載済）。
- `HistogramSeries` はマーカー非対応。ラインかキャンドルに付与する。

## 5. B&H比較・取引コスト控除の共通部品化 ★★

### 改善内容
`CustomReturnChart.tsx:478-493` のバイ&ホールド比較バー（超過リターン・超過年率の色分け）が秀逸。
これを条件付き戦略系の**共通部品 `<StrategyVsBenchmark>`** に昇格させ、全戦略系で再利用。

加えて:
- **取引コスト入力**（手数料・スプレッド bps）を1か所で設定し、全戦略の累積リターンから控除。
  現状ガイドに「コスト未考慮」と注記があるだけ（`:544`）なので、実数で効かせる。
- **回転率（売買頻度）表示**: コスト感度の前提として年間取引回数を併記。

### 注意点
極値ベース戦略（高値買い/安値売り）は執行不能の理論上限。コスト控除後も**実現不可ラベル**を維持。

## 6. 任意時刻ウィンドウ × 曜日のintradayクロス集計 ★

### 改善内容
intraday基盤（`intraday-core` / `useIntraday`）が既にあるので、
「**任意の時刻ウィンドウ（例 9:00–9:30）× 曜日**」のクロス集計UIを追加。
寄り後30分／引け前30分のエッジを任意指定で検証できるようにする。

### 実装の足場
intraday系コンポーネントに、開始/終了時刻ピッカー＋曜日トグルを渡す。
集計は `conditional-forward-returns` と同じ「状態→フォワード」枠組みに時刻条件を足すだけ。

### 注意点
intradayデータは取得期間が短く標本が少ない。**信頼区間とサンプル数の明示が必須**。

## 7. 期間選択の任意化 ★


### 現状の制約
`PeriodSelector.tsx` は `1m〜10y` の8プリセットのみ。`filteredPrices`（`useAnalysisData.ts:52-58`）は
末尾固定スライス。

### 改善内容（※グローバル日付レンジ本体は別タスク。ここでは付随UIのみ）
- **対数スケール/線形スケール切替**: 長期チャートでの%変化把握を容易に。
- **ローリング窓長の任意入力**: 多くの分析が窓長固定（例: realized vol 20日）。
  代表的な窓長パラメータを数値入力化し、感応度を確認できるように。

## 横断的な設計上の注意（重要）

間引き/絞り込み系のフィルタを分析に渡す際、**等間隔・連続性を仮定する分析が壊れる**。

- フィルタOK（リターン集計・分布・条件付き・カレンダー系）と、
  フィルタNG（ACF・GARCH・ウェーブレット・EMD・再帰定量化など時系列構造系）を区別する。
- コンポーネント側に `acceptsFilter: boolean` のような宣言を持たせ、NG系には `allPrices` を渡す。
- 各バケット/ビンに**最小サンプル数の警告**と**有意性バッジ**（既存 `StatBadge`）を必ず併記し、
  少数標本のノイズを「シグナル」と誤読させない。
