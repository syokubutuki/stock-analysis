# 日中足データ分析 — 実装プラン詳細

本書は、**日中足（intraday bars: 分足OHLCV）** を用いて短期トレードの執行タイミングを高度化する分析機能の実装プランを、各項目ごとに「目的 / 入力 / 新規・拡張ファイル / アルゴリズム・数式 / コンポーネント設計 / AnalysisGuide要点 / 注意点・サンプル数 / 優先度」まで掘り下げて記載する。

姉妹文書: `docs/daily-data-analysis-proposals.md`（日足側カタログ）。**棲み分けの原則**: 長期の持続性・断面の質は日足側、**当日内の執行タイミングの質**は本書（日中足側）が担う。

> 規約: 新規コンポーネントは必ず `AnalysisGuide` で手法・数式・用語・読み方・活用・注意点を解説すること（`CLAUDE.md`）。

## 0. 共通基盤（既存資産と制約）

### 0.1 データ取得経路

`app/api/intraday/route.ts`（実装済み）が Yahoo Finance の分足を返す。

| interval | 取得可能な過去期間（Yahoo上限） | 営業日換算の目安 |
|---|---|---|
| `1m` | 7日 | 約5営業日 |
| `5m` | 60日 | 約40営業日 |
| `15m` | 60日 | 約40営業日 |
| `30m` | 60日 | 約40営業日 |
| `60m` | 730日 | 約500営業日 |

レスポンス: `{ symbol, name, interval, range, gmtoffset, timezone, currency, bars: IntradayBar[] }`。
`IntradayBar = { ts: number(UNIX秒UTC), open, high, low, close, volume }`。
**取引所ローカル時刻は `(ts + gmtoffset)` で算出**（`highlow-timing.ts` の `localMinute` / `localDay` / `localDateStr` を再利用）。

### 0.2 取得・状態管理パターン（既存）

`HighLowTimingChart.tsx` が確立済みのパターンを踏襲する:

```tsx
const [intervalKey, setIntervalKey] = useState("5m");
const [resp, setResp] = useState<IntradayResponse | null>(null);
const [loading, setLoading] = useState(false);
useEffect(() => {
  setLoading(true);
  fetch(`/api/intraday?ticker=${encodeURIComponent(ticker)}&interval=${intervalKey}`)
    .then(r => r.json()).then(setResp)
    .catch(e => setError(String(e))).finally(() => setLoading(false));
}, [ticker, intervalKey]);
```

→ **新規共通フック `app/hooks/useIntraday.ts` を切り出す**ことを推奨（現状は `HighLowTimingChart` 内に直書き）。複数の新規コンポーネントが日中足を使うため、取得・キャッシュ・ローディング・エラーを集約する。

```ts
// useIntraday(ticker, interval) → { resp, loading, error }
// 同一(ticker,interval)はモジュールレベル Map でメモ化し再fetchを避ける
```

### 0.3 既存の日中足資産（重複回避のため明記）

| ファイル | 役割 | 流用ポイント |
|---|---|---|
| `app/lib/highlow-timing.ts` | 高安到達時刻・条件付け・OR/前日水準ブレイク・日次レコード抽出 | `IntradayBar`, `DayRecord`, `extractDayRecords`, `buildBinGrid`, `binIndexOf`, `localMinute` 等を **export して再利用** |
| `app/lib/intraday-range.ts` | 正規化レンジ・ローリング・レンジ統計 | 日足ベース。分足版RVと混同しない |
| `app/components/analysis/HighLowTimingChart.tsx` | 高安タイミングUI（足選択・条件・ビュー切替） | 取得/Canvas/足切替UIの雛形 |
| `IntradayPathChart` / `IntradayRangeChart` / `CandleSeasonalityChart` | 経路・レンジ・季節性 | 既存ビューとの重複に注意 |

### 0.4 横断の必須要件（全項目共通）

- **サンプル数 n の明示**: `5m`=約40営業日しかない。時間帯アノマリーは n とともに**ブートストラップ信頼区間 + FDR（多重比較補正）** を付す（`weekday-scan.ts` の手法を踏襲）。
- **遅延の明示**: Yahoo分足は約15分遅延。「現在値」系（B1/A4のライブ判定）は遅延前提のUI表記を必ず入れる。
- **欠損バー耐性**: 昼休み（東証 11:30–12:30）・出来高ゼロ・null OHLC を除外。ビン集計は存在バーのみで平均。
- **取引所セッション非依存設計**: ビン境界はデータ実測の `sessionStart/sessionEnd` から構築（`buildBinGrid` 方式）。米株・東証で時刻が違うためハードコードしない。

### 0.5 共通ユーティリティの新設

`app/lib/intraday-stats.ts`（新規）— 全項目が使う統計基盤を集約:
- `bootstrapCI(values, statFn, B=2000, alpha=0.05)` — ブロックブートCI
- `benjaminiHochberg(pValues, q=0.05)` — FDR
- `tStat(values)` / `welchT(a, b)` — 平均の有意性
- `weightedMean` / `winsorize`（外れ値の寄与抑制）

## A. 時間帯プロファイル（「いつ動くか」）

### A1. 時間帯別ボラ／出来高プロファイル（U字曲線） ★★★

**目的**: 1日の中で「いつボラ・出来高が高いか」を可視化し、約定コストの低い時間帯と動きやすい時間帯を分離する。全執行系の土台。

**入力**: `5m` または `15m` 足（`60m`では粒度不足）。複数日のバーを時刻ビンに畳み込む。

**新規ファイル**:
- `app/lib/intraday-profile.ts`
- `app/components/analysis/IntradayProfileChart.tsx`

**アルゴリズム / 数式**:
時刻ビン *b*（例: 5分刻み、ローカル時刻）ごとに、全営業日 *d* を横断して集計する。

- バー内ボラ代理（Garman-Klass の1バー版）:
  `rv_{d,b} = 0.5·(ln(H/L))² − (2ln2−1)·(ln(C/O))²`
- 各ビンの平均ボラ: `RV(b) = mean_d rv_{d,b}`、年率換算は任意。
- リターン絶対値プロファイル: `AR(b) = mean_d |ln(C_{d,b}/O_{d,b})|`
- 出来高プロファイル: `V(b) = mean_d volume_{d,b}`（日内合計で正規化し相対出来高 `V(b)/Σ_b V(b)` を併記）
- スプレッド代理（Corwin-Schultz の隣接2バー版を時刻ビンで）: 取引コストの時刻依存を近似。

各プロファイルにブートCIのバンドを付す。

**コンポーネント設計**:
- Canvas（`initCanvas`パターン）で横軸=時刻、縦軸=値の折れ線3本（ボラ/出来高/スプレッド代理）をトグル表示。
- 足選択（5m/15m/30m）、対象期間（直近20日/40日）、曜日フィルタ（任意）。
- ビン値ホバーで n・平均・CI をツールチップ表示。
- 典型的なU字（寄り・引け高、昼安）を視覚化。

**AnalysisGuide要点**: U字曲線の意味、GKボラ1バー版の数式と導出、相対出来高の定義、「寄り/引けは高ボラ＝スリッページ大、約定は中間帯が有利」という読み方、遅延とサンプル数の注意。

**注意点**: 東証は前場/後場の2セッション → 昼休みをまたぐビンを欠損として扱う。`includePrePost=false` 前提。

### A2. 時刻別の方向ドリフト（日内アノマリー） ★★

**目的**: 「午前は上がりやすい／後場頭は弱い」等、時刻に紐づく符号付きドリフトを有意性込みで検出。

**入力**: A1と同じ畳み込み。リターンの**符号**を見る点が違う。

**新規ファイル**: `app/lib/intraday-profile.ts` に `directionalDrift()` を追加（A1と同居）、ビューはA1コンポーネントのタブとして実装。

**アルゴリズム / 数式**:
- 各ビンの平均対数リターン: `μ(b) = mean_d ln(C_{d,b}/O_{d,b})`
- t統計: `t(b) = μ(b) / (s(b)/√n)`、p値 → **BH-FDR** で多重比較補正。
- 累積日内ドリフト曲線: `Cum(b) = Σ_{b'≤b} μ(b')`（平均的な「1日の形」）。
- 勝率（プラス確率）と平均/中央値を併記。

**コンポーネント設計**: 棒グラフ（ビンごとμ、有意なビンを濃色）＋累積ドリフト折れ線。曜日別の小倍率（small multiples）切替。

**AnalysisGuide要点**: ドリフトとボラの違い、FDRの必要性（ビン数だけ多重検定になる）、サンプル外での消滅可能性、執行タイミングへの落とし込み。

**注意点**: n小のため過剰解釈厳禁。CI・FDR・年次（前半/後半）再現性を必須表示。

### A3. オープニングレンジ(OR)統計の拡張 ★★

**目的**: 寄り後 *k* 分のレンジ（OR）を基準に、上下ブレイクの方向・追随率・だまし率・引け到達位置を測り、ORブレイク戦略を最適化。`highlow-timing.ts` に一部あるOR概念を独立分析へ昇格。

**入力**: `1m`〜`5m`（寄り直後の解像度が要るため細かい足が望ましい）。

**新規ファイル**:
- `app/lib/opening-range.ts`（`extractDayRecords` を流用）
- `app/components/analysis/OpeningRangeChart.tsx`

**アルゴリズム / 数式**:
- OR定義: 寄り後 *k* 分 ∈ {5,15,30,60} の高値 `ORH`、安値 `ORL`、幅 `ORW = ORH−ORL`。
- 上抜けイベント: ある後続バーで `high > ORH`。上抜け後に **引けまで `ORH` を維持**した割合 = 追随率、上抜け後に `ORL` を割った割合 = 反転（だまし）率。
- 期待値: `E[ret | 上抜け] = mean(close/breakPrice − 1)`（下抜けも対称に）。
- ORW を当日ボラ（前日ATR等）で正規化し、「広いOR/狭いOR」別に成績を層別。
- 目標到達: ブレイク後に `1×ORW`, `2×ORW` 拡張へ到達した割合（利確目標の妥当性）。

**コンポーネント設計**: *k* スライダ、ブレイク方向別の追随/だまし率バー、ブレイク後経路のスパゲッティ＋中央値、ORW分位別成績テーブル（n付き）。

**AnalysisGuide要点**: ORブレイク戦略の概要、追随率/だまし率の定義、ORW正規化の理由、利確目標(R倍)設定への活用、薄商い銘柄での不安定性。

### A4. VWAP乖離の回帰・継続統計 ★★★

**目的**: 当日VWAPからの乖離が「戻る（平均回帰）」か「伸びる（トレンド）」かを統計化し、押し目買い・乖離過大での利確の根拠を与える。日計りで最も実戦的。

**入力**: `5m`（VWAPは出来高加重のため出来高必須）。

**新規ファイル**:
- `app/lib/vwap-analysis.ts`
- `app/components/analysis/VwapDeviationChart.tsx`

**アルゴリズム / 数式**:
- 当日 typical price `TP = (H+L+C)/3`、累積VWAP: `VWAP_t = Σ_{i≤t}(TP_i·V_i) / Σ_{i≤t}V_i`（各営業日で寄りからリセット）。
- 乖離: `dev_t = (C_t − VWAP_t)/VWAP_t`。
- 標準化: 当日内の乖離標準偏差 `σ_dev` で `z_t = dev_t/σ_dev`（VWAPバンドに相当）。
- **条件付き先行リターン表**: `z_t` を分位バケット（例: <−2, −2..−1, −1..0, 0..1, 1..2, >2）に分け、各バケットの *h* バー先リターン `ln(C_{t+h}/C_t)` の平均・勝率・分散を集計（回帰か継続かの判定）。
- VWAP上抜け/下抜け後の追随率も併記。

**コンポーネント設計**: 上段=ある代表日のVWAP±バンドと価格、下段=zバケット→先行リターンのヒートマップ/バー（h スライダ）。足・期間選択。

**AnalysisGuide要点**: VWAPの意味（機関投資家の平均約定）、乖離Zとバンドの数式、回帰/継続の読み分け、「Z<−1で買い候補／Z>+2で利確」の使い方、トレンド日では回帰が効かない点。

**注意点**: VWAPは日次リセット。前場/後場で連続。出来高ゼロバー除外。15分遅延でライブ運用には不向きと明記。

## B. 当日内の経路・状態（「今日はどういう日か」）

### B1. 日内トレンド／レンジ判定 ★★★

**目的**: 当日が「トレンド日」か「レンジ日」かを当日内で更新判定し、順張り/逆張りの戦略スイッチにする。

**入力**: `5m`。当日の累積バー列。

**新規ファイル**:
- `app/lib/intraday-regime.ts`
- `app/components/analysis/IntradayRegimeChart.tsx`

**アルゴリズム / 数式**:
- Kaufman効率比（当日内、寄りから時刻*t*まで）:
  `ER_t = |C_t − O_first| / Σ_{i≤t}|C_i − C_{i-1}|`（1に近いほどトレンド、0に近いほどノイズ/レンジ）。
- 分散比 VR(q): `VR(q) = Var(r^{(q)}) / (q·Var(r^{(1)}))`（>1トレンド、<1平均回帰）を日内リターンで。
- 終値のレンジ内位置 `closePos`（`DayRecord` 既存）と組合せて「トレンド日 = ER高 & closePos極端」を分類。
- 各営業日を {強トレンド上/強トレンド下/レンジ} にラベル付け → ラベル別の翌日寄りギャップ・翌日リターン分布を集計（前日のレジームが翌日に効くか）。

**コンポーネント設計**: 当日のER_t時系列（閾値線付き）、日別レジームの帯（カレンダーストリップ）、レジーム別の翌日成績テーブル。

**AnalysisGuide要点**: 効率比/分散比の数式と直感（「まっすぐ進んだ距離 ÷ 歩いた総距離」）、トレンド日/レンジ日の戦略対応表、当日確定前の判定は暫定である注意。

### B2. 寄り→現在の経路類似マッチング（日中アナログ） ★★

**目的**: 当日の前場波形に似た過去日を検索し、その後場・引けの分布を提示。「この立ち上がりの日は引けが高い/安い」を直感的に示す。

**入力**: `5m`。当日の寄り〜現時点の正規化経路。

**新規ファイル**:
- `app/lib/intraday-analog.ts`
- `app/components/analysis/IntradayAnalogChart.tsx`

**アルゴリズム / 数式**:
- 当日経路を寄り基準で正規化: `p_t = (C_t − O_first)/O_first`（または日中ボラで標準化）。
- 過去各営業日の同区間ベクトルと **DTW距離**（または時刻整列済みユークリッド）で類似度算出。
- 近傍 *K* 日（最小距離）を抽出し、その後場〜引けの経路を重ね描き、引けリターン `dayRet` の分布（平均/中央/勝率/分位）を集計。
- 距離による重み付き予測（近いほど重い）。

**コンポーネント設計**: 当日経路（太線）＋近傍K日の続き（細線、半透明）、引けリターン分布ヒストグラム、K・区切り時刻（前場終/任意）スライダ。

**AnalysisGuide要点**: DTWの概念（伸縮を許す波形距離）、正規化の意味、近傍予測の前提（過去の繰り返し性）、サンプル小・自己相似の罠、「占い化」しないための分布提示の重要性。

**注意点**: `5m`×40日では近傍プールが小さい。`60m`で長期プールを併用する設計を検討。

### B3. 前場→後場の予測関係 ★★

**目的**: 昼休み時点の情報（前場リターン・前場レンジ内位置・前場出来高）から後場リターンを条件付けし、持ち越し/手仕舞い判断を支援。東証の2セッション構造を活用。

**入力**: `5m`。前場（〜11:30）/後場（12:30〜）を分割。

**新規ファイル**:
- `app/lib/session-split.ts`
- `app/components/analysis/SessionSplitChart.tsx`

**アルゴリズム / 数式**:
- 前場リターン `r_AM = (C_{11:30} − O_open)/O_open`、後場リターン `r_PM = (C_close − O_{12:30})/O_{12:30}`。
- 前場レンジ内の引け位置、前場/全日出来高比などを説明変数化。
- 条件付き表/散布図: `r_AM` 分位 → `r_PM` の平均・勝率・相関（継続か反転か）。
- 単回帰 `r_PM = α + β·r_AM + ε` の係数・有意性・R²。

**コンポーネント設計**: `r_AM` vs `r_PM` 散布図（回帰直線・相関）＋分位バケット表。米株（単一セッション）では「午前/午後」の任意分割にフォールバック。

**AnalysisGuide要点**: 2セッション構造の説明、継続/反転の読み方、回帰係数の解釈、昼の手仕舞い判断への活用、セッションのない市場での扱い。

### B4. 当日内 MFE/MAE（最大含み益・含み損の時刻分布） ★★★

**目的**: 寄り（または任意時刻）でエントリーした場合の当日内 最大含み益(MFE)/最大含み損(MAE)とその到達時刻を測り、日計りのTP/SL・時間切り(time stop)を最適化。日足版 `MFEMAEChart` の日内版。

**入力**: `5m` または `1m`（経路精度が要る）。

**新規ファイル**:
- `app/lib/intraday-excursion.ts`
- `app/components/analysis/IntradayExcursionChart.tsx`

**アルゴリズム / 数式**:
- エントリー時刻 *e*（寄り/OR確定後/任意ビン）、方向（ロング/ショート/当日符号）を指定。
- 各日: `MFE = max_{t>e}(C_t − E)/E`、`MAE = min_{t>e}(C_t − E)/E`、各到達時刻 `t_MFE, t_MAE`。
- 分布: MFE/MAEのヒストグラム、到達時刻のヒストグラム、`E[終値リターン | MAEが −x% に達した]`（損切り水準別の期待値曲線）。
- **TP/SL最適化**: 利確 `tp`・損切り `sl` のグリッドで、当日経路にルール適用したときの期待値・期待R・勝率をヒートマップ化（時間切りも変数に）。

**コンポーネント設計**: エントリー時刻/方向セレクタ、MFE/MAE分布、TP×SLヒートマップ（期待値）、最適点ハイライト。

**AnalysisGuide要点**: MFE/MAEの定義、R倍数、TP/SLグリッド探索の意味、過剰最適化（in-sampleでの最良点）への警告とブロックブート頑健性、時間切りの効用。

**注意点**: ルックアヘッド注意（同一バー内でTPとSL双方ヒット時は保守的にSL優先など規約を明記）。

## C. マイクロ構造の代理（板なしでできる範囲）

### C1. 分足版ボラ推定 → 実現ボラ(RV)・実現レンジ ★★★

**目的**: 分足から日次の実現ボラを高精度に推定し、当日ボラの割高/割安判定とHARボラ予測の入力にする。日足のσ推定より分散が小さい。

**入力**: `5m`（RVの定番サンプリング）。日次に集計。

**新規ファイル**:
- `app/lib/realized-vol.ts`
- `app/components/analysis/RealizedVolChart.tsx`

**アルゴリズム / 数式**:
- 日次実現分散: `RV_d = Σ_i r_{d,i}²`（`r_{d,i}` = 5分対数リターン）、実現ボラ `√RV_d`、年率 `√(252·RV_d)`。
- ノイズ頑健版: 実現レンジ `RR_d = Σ_i (ln H_i − ln L_i)²/(4ln2)`、二乗和バイアスにはサブサンプリング平均を任意。
- バイパワー変動 `BV_d`（ジャンプ除去）と `RV_d − BV_d` で**ジャンプ成分**を分離。
- **HAR-RV予測**: `RV_{d+1} = β0 + βd·RV_d + βw·RV_d^{(week)} + βm·RV_d^{(month)} + ε`（OLS、係数・R²・残差診断）。
- オーバーナイト分散（夜間ギャップ²）を別建てし全日ボラへ合算。

**コンポーネント設計**: RV/RR/BVの時系列、ジャンプ日マーカー、HAR予測 vs 実測の重ね描き＋予測区間、係数表。

**AnalysisGuide要点**: 実現ボラの概念（高頻度で測った真のボラ）、RV/RR/BVの数式、ジャンプ分離の意味、HARモデルの直感（日/週/月の記憶）、夜間ボラの扱い、マイクロストラクチャ・ノイズ（5分が定番の理由）。

---

### C2. オーバーナイト vs 日中の精密分解 ★★

**目的**: 「夜間ギャップ」と「日中ザラ場」のリターン・リスク寄与を分足で正確に分離（日足近似より精密）。持ち越しリスクと日中ドリフトを切り分ける。

**入力**: `5m` または `60m`（長期は60m）。

**新規ファイル**: `app/lib/realized-vol.ts` に同居 or `app/lib/overnight-intraday.ts`、ビューは `RealizedVolChart` のタブ。

**アルゴリズム / 数式**:
- 夜間（オーバーナイト）リターン: `r_ON = ln(O_open / C_prevClose)`。
- 日中リターン: `r_ID = ln(C_close / O_open)`。
- 累積エクイティ比較: 「引け買い→翌寄り売り（夜間取り）」vs「寄り買い→引け売り（日中取り）」の累積曲線。
- リスク寄与: `Var(r_ON)/Var(r_total)`、テールの夜間集中度（夜間が大幅下落日に占める割合）。

**コンポーネント設計**: 2本の累積エクイティ、夜間/日中の分散・シャープ・最大DD比較表、テール集中度。

**AnalysisGuide要点**: 分解の定義、オーバーナイト・ドリフト異常、持ち越しリスクの定量化、執行時間帯の選択（どちらの時間に張るべきか）。日足版（`daily 8.1`）との精度差を明記。

---

### C3. 日中の自己相関・シグネチャープロット ★

**目的**: 日中リターンのラグ別自己相関と、サンプリング間隔によるRVの変化（シグネチャープロット）から、ノイズ帯・最適売買間隔を把握。

**入力**: `1m`（最細）〜`30m` を横断。

**新規ファイル**:
- `app/lib/intraday-autocorr.ts`
- `app/components/analysis/IntradaySignatureChart.tsx`

**アルゴリズム / 数式**:
- 日中リターンの自己相関 `ρ(k)`（日跨ぎは除外、各日内で算出して平均）。Bartlett帯で有意性。
- シグネチャープロット: サンプリング間隔 Δ ∈ {1,2,5,10,15,30}分 ごとの平均 `RV(Δ)` をプロット。Δ→0でRVが発散（増加）するならマイクロストラクチャ・ノイズの存在を示す。
- 最適サンプリング間隔の示唆（プロットが平坦化する点）。

**コンポーネント設計**: ρ(k)バー（有意帯）＋シグネチャープロット折れ線。`1m`取得（7日）と`5m`の併用。

**AnalysisGuide要点**: 自己相関の意味、ビッドアスク・バウンスによる負の1次自己相関、シグネチャープロットでノイズを見る方法、RV推定の間隔選択への含意。

**注意点**: `1m`は7日のみ → ρ推定のn極小。注意書き必須。

---

### C4. 出来高クロック（タイムバー → ボリュームバー） ★

**目的**: 等時間バーを等出来高バーに再標本化し、イベント時間（情報量）ベースで分析。ニュース急変・出来高集中の検出に有効。

**入力**: `5m`（volume必須）。

**新規ファイル**:
- `app/lib/volume-clock.ts`
- `app/components/analysis/VolumeClockChart.tsx`

**アルゴリズム / 数式**:
- 1日の総出来高を *N* 分割し、累積出来高が `(k/N)·V_total` を超えるごとに1ボリュームバーを確定。
- ボリュームバー上でのリターン分布は時間バーより正規に近づく（分散の時間依存が緩和）。これを QQ/分散比で確認。
- 出来高加速度（単位時間あたりボリュームバー生成数）でイベント検出。

**コンポーネント設計**: 時間バー vs ボリュームバーのリターン分布比較（QQ・ヒスト）、ボリュームバー生成密度の時刻プロファイル。

**AnalysisGuide要点**: タイムバー/ティック/ボリューム/ドルバーの違い、情報時間の概念、正規性改善の意味、イベント検出への活用（López de Prado 系の発想）。

---

## D. 日足×日中の連結（既存資産と直結）

### D1. ギャップ後の日中挙動 ★★

**目的**: 寄りギャップの「埋め」時刻、gap-and-go（継続）か fade（窓埋め）かを分足で実測。日足の `GapAnalysisChart` を日中で裏取り。

**入力**: `5m`。`extractDayRecords` の `gap` フィールド流用。

**新規ファイル**:
- `app/lib/gap-intraday.ts`
- 拡張 or 新規 `GapIntradayChart.tsx`（`GapAnalysisChart` と連携明記）

**アルゴリズム / 数式**:
- ギャップ日を上窓/下窓・大小で分類（`gap` を前日ボラで正規化）。
- 窓埋め: 当日中に `prevClose` 水準へ到達した割合と**到達時刻分布**。
- gap-and-go: 寄り後 *k* 分でギャップ方向に伸びた割合 → 引けでの継続率。
- ギャップ区分別の日中経路（スパゲッティ＋中央値）と引けリターン。

**コンポーネント設計**: ギャップ区分セレクタ、窓埋め時刻ヒスト、継続/fade率、区分別経路。

**AnalysisGuide要点**: 窓の分類、gap-and-go vs fade、窓埋め時刻の活用（寄り直後の戦略選択）、薄商い銘柄の窓の信頼性。

---

### D2. 曜日 × 時間帯ヒートマップ（日内版 weekday-scan） ★★

**目的**: 「金曜後場は弱い」等、曜日×時刻の複合アノマリーを好機スキャン。`weekday-scan.ts` の手法を日内に展開。

**入力**: `5m`/`15m`。

**新規ファイル**:
- `app/lib/weekday-intraday-scan.ts`（`weekday-scan.ts` の統計手法を再利用）
- `app/components/analysis/WeekdayIntradayHeatmap.tsx`

**アルゴリズム / 数式**:
- セル = (曜日 w, 時刻ビン b)。値 = `mean ln(C/O)`（または勝率）。
- 各セルに t統計 → **BH-FDR**（5曜日 × ビン数の多重比較）。
- 安定性: 前半/後半データでの符号一致、ブートCI。
- エッジの大きさ × 有意性で色分け。

**コンポーネント設計**: 曜日（行）× 時刻（列）ヒートマップ、有意セルに枠、ホバーで n/平均/CI/p。`weekday-scan` の素片分解UIに準拠。

**AnalysisGuide要点**: 多重比較の罠（曜日×時刻でセル数膨大）、FDRと安定性検証の必須性、複合エッジの活用、賞味期限（年次持続）。

**注意点**: n が最も枯渇する分析。セルあたり最小サンプル閾値（例: n≥10）を設けて非表示にする。

---

### D3. 日足シグナル発生日の翌日寄り後の動き ★★

**目的**: 日足の状態（RSI過熱/ブレイク/ギャップ）が出た翌日、寄り後どの時刻にエントリーするのが最適かを日中足で測る。日足シグナルと日内執行の橋渡し。

**入力**: 日足（`/api/stock`）でシグナル日を抽出 → 該当翌日の日中足（`/api/intraday`）。

**新規ファイル**:
- `app/lib/signal-intraday.ts`
- `app/components/analysis/SignalIntradayChart.tsx`

**アルゴリズム / 数式**:
- 日足シグナル集合 S（例: RSI(14)<30, ドンチャン上抜け, 大陽線翌日）を日足から生成。
- シグナル翌日の日中経路を寄り基準で正規化し重ね描き。
- 「寄り成り / OR確定後 / 前場安値 / VWAP回帰」など複数のエントリールール別に、当日内の期待リターン・勝率を比較。
- 最適エントリー時刻（期待値最大の時刻ビン）を提示。

**コンポーネント設計**: シグナル種セレクタ（日足）、翌日経路スパゲッティ＋中央値、エントリールール別成績表。

**AnalysisGuide要点**: 日足→日内のシグナル連結、エントリータイミングの期待値比較、ルックアヘッド回避、シグナル日数（n）の制約。

**注意点**: シグナル翌日が `60d`（5m）窓内に十分数あるか n を明示。長期は `60m` を使用。
